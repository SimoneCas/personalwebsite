Stream vs Parallel Stream

Giugno 13, 2019

Per lavorare su collezioni di dati effettuando operazioni di calcolo, riduzione e aggregazione ci può risultare utile l'utilizzo degli Stream e dei Parallel Stream.

L'esecuzione di uno Stream avviene in un unico thread, gestendo in modo sequenziale le operazioni.
L'esecuzione di un Parallel Stream avviene invece utlizzando molteplici thread, senza mantenere l'ordinamento degli elementi elaborati.
Esempio di Stream: 

numbers.stream()
	.map(e -> transform(e))
	.forEach(System.out::println);
Esempio di Parallel Stream 
numbers.parallelStream()
	.map(e -> transform(e))
	.forEach(System.out::println);

or

numbers.stream()
	.parallel()
	.map(e -> transform(e))
	.forEach(System.out::println);
Attenzione: l'utilizzo del metodo "sequential()" forzando l'esecuzione del Parallel Stream in modo sequenziale non permette di eseguire elaborazione parallela.
Se si vuole utilizzare il Parallel Stream per ottimizzare l'elaborazione ma si vuole mantenere ordinati i valori originali, si può utilizzare il metodo "forEachOrdered()" 
numbers.stream()
	.parallel()
	.map(e -> transform(e))
	.forEachOrdered(System.out::println);
La configurazione del numero di thread da poter assegnare al parallelStraem è importante sia per gestire l'applicazione in modo sicuro rispetto al numero di cores e sia per configurarla al meglio in base alla tipologia di lavoro che lo stream dovrà svolgere.
Regola base di configurazione:

		# of cores
# thread <= ----------------------------------
		1 - blocking factor

0 <= blocking factor <= 1
dove:
0 se i thraed sono CPU intensive: task che prevedono pochi interrupt ed elaborazioni lunghe ed intensive lato cpu
1 se i thread prevedono molti interrupt, dovuti ad esempio ad interazioni con sistemi esterni, e che prevedono elaborazioni brevi lato cpu.
Il numero di thread da utilizzare per eseguire l'applicazione può essere configurato:
  • A livello di jvm: -Djava.util.concurrent.ForkJoinPool.common.parallelism=#Thread
  • A livello programmatico:
ForkJoinPool pool = new ForkJoinPool(50);
pool.submit(() ->
	numbers.stream().parallel()
		.map(e -> transform(e))
		.forEach(e -> {})
);

pool.shutdown();
pool.awaitTermination(10, TimeUnit.SECONDS);
Quando conviene lo stream parallelo?
  • Quando abbiamo una mole di dati da elaborare di grandi dimensioni
  • Per task che non richiedano parecchie risorse di tempo e non producano eccezioni
