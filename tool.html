JVM Monitoring

Sviluppando progetti su JVM è indispensabile monitorare e configurare temi di performance della JVM sulla quale è avviata l'applicazione. Risulta fondamentamentale quindi monitorare la JVM sia a runtime che a posteriori.
Per analisi a runtime tramite protocollo JMX - Java VisualVM: permette di monitorare l'utilizzo di memoria, cpu, thread e offre la possibilità di installare plug-in per analisi di dettaglio.

Per analizzare invece a posteriori il comportamento di una JVM - Java Flight Recorder + Java Mission Control: tramite JFR è possibile salvare su file le statistiche della JVM in modo automatizzato così da poterle analizzare in un secondo momento tramite il tool Java Mission Control.

Garbage Collector analyzer

Sviluppando progetti su JVM è indispensabile monitorare e configurare temi di performance dell'applicazione. Risulta fondamentamentale analizzare le attività del Garbage Collector per analizzare l'utilizzo dell'heap.
Tool di analisi online del file di log xxx.gc generato dalla JVM:
GC-easy

Se si vuole invece utilizzare un tool il locale:
GC Viewer
